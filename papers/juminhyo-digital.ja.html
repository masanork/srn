検討資料：住民票の写し等の電子交付における要件定義と Web/A による実現方式

2024年12月25日
構成員：楠 正憲

1. はじめに

総務省「デジタル技術を活用した効率的・効果的な住民基本台帳事務等のあり方に関するワーキンググループ」における「中間とりまとめ」（2025年6月）では、住民票の写し等の電子交付について、利便性の向上と事務の効率化、そして原本性の確実な担保が強く求められている。

本資料では、先のホワイトペーパーで提案したアーカイブ品質のWeb文書フォーマット 「Web/A」 を前提とし、中間とりまとめで示された要件をいかに技術的に解決し、社会実装可能なものとするかについて、具体的な実現方式を提案する。

2. 中間とりまとめにおける主要要件の整理

住民票の電子交付にあたっては、従来の紙の証明書が有していた「信頼」をデジタル上で再構成する必要がある。中間とりまとめ、および関連する議論において重視されている要件は以下の4点に集約される：

  1. 原本性の確保（改ざん検知）: 文書の内容が発行時から一文字も変更されていないことを暗号的に証明できること。
  2. 人間と機械の解釈の同一性: 人間の目で見た内容（表示）と、システムが処理するデータ（構造化データ）が完全に一致していること。
  3. 所持者結合（Holder Binding）: 証明書と、その提示者が正当な本人であることを紐付け、スクリーンショット等の単純なコピーによる「なりすまし」を防止すること。
  4. 検証の容易性と自己完結性: 特殊な専用アプリや発行元のサーバーへの常時問い合わせを必要とせず、オフラインや民間窓口でも即座に真正性を確認できること。

3. Web/A フォーマットによる技術的解決策

Web/A は、以下の技術的特徴により、上記の要件を Web 標準の延長線上で解決する。

3.1. 原本性の確保と HMP (Human-Machine Parity)

Web/A 文書は、表示のための HTML と、データ処理のための JSON-LD を単一のファイル内で密結合させる。

  • HMP 署名: 生成ツールは、HTML コンテンツのハッシュ値と JSON-LD データを一括して署名対象に含める。これにより、PDF においてしばしば問題となる「表示内容と署名対象データの乖離（Visual Masking Attack）」を構造的に排除する。
  • ハイブリッド署名: 古典暗号（Ed25519）と耐量子暗号（ML-DSA-44）を組み合わせて署名し、将来的な量子コンピュータの脅威からも文書を保護する。

3.2. Passkey を活用した動的 Holder Binding

中間とりまとめで示された「なりすまし防止」の鍵は、文書そのものに 「所持者の提示意思」 を動的に付与できるかにある。

  • オンデバイス署名: Web/A 文書の Viewer レイヤーに検証ロジックを内蔵する。提示の際、ユーザーがスマホの Passkey (WebAuthn) で認証することで、その瞬間限定の「提示署名（Verifiable Presentation）」をブラウザ内で生成・付与する。
  • 専用アプリ不要: この処理は標準的なモバイルブラウザ上で動作するため、住民に特定アプリのインストールを強いることなく、高度な所持者結合を実現できる。

3.3. 高い暗号アジリティと持続性

住民基本台帳事務は極めて長期の継続性が求められる。

  • DID (Decentralized Identifier) による信頼: DNS や WebTrust 認定の TLS 鍵を起点としつつ、DID を用いて発行鍵を管理する。これにより、将来のアルゴリズム移行（暗号アジリティ）の際も、発行者のアイデンティティを維持したまま、安全に最新の暗号方式へ「洗い替え」を行うことが可能である。

4. 住民票電子交付の運用モデル案

4.1. 発行フロー（自治体）

  1. データ生成: 基幹システムから住民票データを抽出。
  2. Web/A ビルド: 自治体内のビルドサーバーが Web/A-1p 準拠の HTML+JSON-LD を生成。
  3. 発行体署名: 自治体のアイデンティティ（DID）に基づく電子署名を付与し、住民のマイナポータル等へ送付。

4.2. 提示・検証フロー（民間窓口・金融機関等）

  1. 提示: 住民がブラウザで Web/A 文書を開き、窓口でスマホ画面を提示。
  2. 検証ボタンの実行: 文書上の「真正性を確認」ボタンを押し、Passkey で認証。
  3. 真正性の証明: 画面上に「発行者：〇〇市」「整合性：正常」等の検証結果が動的に表示される。
  4. データ取得: 窓口側がデータの提出を求める場合、Web/A 文書から JSON-LD 部分を直接抽出し、基幹システムへ自動取り込みを行う。

5. 提言：今後の検討に向けた方向性

本ワーキンググループにおいては、単なる「紙のPDF化」に留まらない、「機械が処理でき、人間が疑いなく信じられる」 デジタル公文書の標準仕様を確立すべきである。Web/A は、以下の 3 点において、電子政府の基盤として極めて有効な候補となると確信している。

  • オープンスタンダードへの立脚: 特定ベンダーの独自技術に依存しない。
  • アクセシビリティと信頼の両立: セマンティック HTML による高いアクセシビリティ。
  • 将来の脅威への備え: PQC 対応と暗号アジリティによる、100 年単位の保存への耐性。

本提案の方式が、効率的かつ効果的な住民基本台帳実務の実現、引いてはデジタル社会の信頼基盤の構築に寄与することを期待する。
